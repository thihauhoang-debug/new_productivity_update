❔ Hướng dẫn đọc dashboard

✕ Đóng

1. KPI row (đầu trang)

Tổng nhân sự (hợp Pick ∪ Pack, không đếm trùng), số người trùng cả 2 task, % người đạt mốc Xanh theo Sản lượng cả ca (ngưỡng chỉnh ở ⚙ Cấu hình), và % Indirect (tỉ lệ lượt người-giờ không phát sinh sản lượng).

🕘 Bộ lọc khung giờ (dùng chung cho cả dashboard)

Chọn "Từ giờ – Đến giờ" ở panel trên cùng (hỗ trợ ca xuyên đêm, VD 22h→04h; chọn "Từ giờ" = "Đến giờ" để soi đúng 1 khung giờ). Toàn bộ dashboard — KPI row, RAG, các bảng tổng hợp, Tổng hợp theo Captain, KPI Hourly, Chi tiết Indirect theo người, Chi tiết theo khung giờ — đều tự tính lại Sản lượng/Direct-Indirect theo đúng khung giờ đang chọn. Bấm "↺ Cả ca" để xem lại toàn bộ 24 giờ.

1-2. Hai bảng tổng hợp: Tổng nhân sự · Đạt target

Mỗi bảng có dòng Tổng (grand-total) và nút tải Excel riêng. "Đạt target" tính theo mốc Xanh của Sản lượng cả ca.

3. Direct/Indirect & Theo Captain

Gộp chung 1 khung, 2 bảng nhỏ cạnh nhau. Trái — Direct/Indirect tổng quan: tính theo lượt người-giờ (mỗi người × mỗi giờ trong ca là 1 lượt; Direct = giờ đó có phát sinh sản lượng). Phải — Theo Captain: gom theo cột Captain (Pick + Pack), cho mỗi Captain số người quản lý, số người đạt target và % đạt target (người chưa gán Captain gộp vào "(chưa gán)"). Bấm vào tên Captain để lọc nhanh bảng "📡 KPI Hourly" và "🔍 Chi tiết Indirect theo người" bên dưới theo đúng Captain đó.

4. Chi tiết theo khung giờ

Với mỗi giờ trong khung đang chọn ở bộ lọc chung phía trên (Pick và Pack tách riêng): số người Direct/Indirect, và số người Đạt/Dưới Target giờ (ngưỡng "Target theo khung giờ" chỉnh ở ⚙ Cấu hình). Dòng Tổng tự tính lại theo đúng khoảng đã chọn, và nút tải Excel 2 sheet (khớp đúng khoảng giờ đang xem).

📡 KPI Hourly (bảng chi tiết theo người)

Cột: #, Email (kèm tag PICK/PACK), Staff Name, Station ID/Name, Captain (sửa được), TOTAL (= tổng sản lượng mọi giờ trong file, tô nền xanh nhạt), rồi các cột giờ — chỉ hiện những giờ thực sự có dữ liệu trong file đang xem, xếp giảm dần. Dòng Total trên cùng luôn cộng dồn theo toàn bộ danh sách đang lọc (không chỉ trang đang xem). Bảng phân trang 20 dòng/trang — dùng thanh điều hướng trang phía dưới bảng để chuyển trang, số thứ tự # tính liên tục qua các trang.

Màu ô theo giờ: xanh = đạt Target giờ, vàng = 70–99% Target, đỏ = dưới 70% Target — Target lấy theo đúng Task (Pick/Pack) của dòng đó, chỉnh ở ⚙ Cấu hình mục "Target theo khung giờ".

Bộ lọc/sắp xếp: tìm theo email, lọc Task, lọc theo Captain, sắp xếp TOTAL cao↔thấp, hoặc bấm tiêu đề bất kỳ cột nào (kể cả từng cột giờ) để sắp xếp theo đúng cột đó. Các cột giờ hiển thị và TOTAL đều theo đúng khung giờ đang chọn ở bộ lọc chung phía trên (chọn "Từ giờ" = "Đến giờ" để soi chính xác 1 khung giờ, kể cả khi giờ đó không ai phát sinh sản lượng).

Thanh công cụ trên bảng: － 100% ＋ phóng to/thu nhỏ chữ trong bảng cho dễ nhìn (tự nhớ lần sau); 🔒/🔓 Chế độ sửa — bật lên mới sửa được Station ID/Station Name/Captain trực tiếp trong bảng (để tránh sửa nhầm khi chỉ xem); ☰ Cột hiển thị — ẩn/hiện Staff Name, Station ID, Station Name, Captain nếu muốn bảng gọn hơn.

Sửa Station ID/Station Name/Captain của 1 người sẽ được nhớ theo email và tự áp lại cho các lần tải file sau, kể cả khi file gốc không có sẵn 3 cột đó.

🔍 Chi tiết Indirect theo người

Mục tiêu là soi nhanh từng người Indirect ở đúng những khung giờ nào. Bảng có bộ lọc Task/Captain riêng (độc lập với bảng "📡 KPI Hourly" phía trên) — chỉ dùng chung ô tìm email và thứ tự sắp xếp. Phân trang 20 dòng/trang như KPI Hourly.

Logic: với mỗi cột giờ trong khung đang chọn ở bộ lọc chung — nếu người đó có sản lượng > 0 thì hiện số; nếu ô trống hoặc = 0 thì hiện Indirect (chữ đậm, màu đỏ). Cột cuối "Số giờ Indirect" đếm tổng số giờ Indirect trên tổng số giờ đang xem, giúp so sánh nhanh giữa các người.

⤢ Toàn màn hình: mở bảng này (kèm 2 bộ lọc + phân trang) trong 1 popup lớn hơn để xem/lọc thoải mái, đặc biệt hữu ích khi có nhiều cột giờ. Đóng bằng nút ✕, bấm ra ngoài popup, hoặc phím Esc.

🕘 Lịch sử tải lên (local, trong trình duyệt)

"Xem lại" nạp đúng dữ liệu thô của lần đó vào dashboard (có banner cảnh báo + nút quay về bản mới nhất); "Tải raw" xuất ra file Excel. Có thể đặt tự động xoá bớt các bản cũ hơn N ngày ở ⚙ Cấu hình (mặc định tắt, giữ vô hạn).

4. Dán URL đó vào ô "Apps Script Web App URL" trong ⚙ Cấu hình → Lưu cấu hình.

Code Apps Script (copy nguyên đoạn, thay cho code cũ): 
function doPost(e) {
  // Nếu bạn bấm nút "Run" trong editor để test, e sẽ là undefined —
  // đây không phải lỗi, chỉ là doPost cần được gọi bằng 1 request POST thật.
  // Dùng hàm testDoPost() bên dưới để test tay, hoặc deploy rồi tải file thử trên dashboard.
  if (!e || !e.postData) {
    return ContentService.createTextOutput(JSON.stringify({status:'error', message:'Không có POST data — hãy gọi qua testDoPost() hoặc gửi request POST thật, đừng bấm Run trực tiếp trên doPost.'}))
      .setMimeType(ContentService.MimeType.JSON);
  }
  var FOLDER_ID = 'DÁN_FOLDER_ID_VÀO_ĐÂY';
  var folder = DriveApp.getFolderById(FOLDER_ID);
  var data = JSON.parse(e.postData.contents);
  // Mỗi lần tải là 1 file riêng, không tìm/đè file cũ -> lưu lịch sử không giới hạn.
  var safeTime = (data.timeStr || '00-00-00').replace(/:/g, '-');
  var fileName = 'productivity_' + data.task + '_' + data.dateStr + '_' + safeTime + '.json';
  var content = JSON.stringify(data, null, 2);
  folder.createFile(fileName, content, MimeType.PLAIN_TEXT);
  return ContentService.createTextOutput(JSON.stringify({status:'ok', file:fileName}))
    .setMimeType(ContentService.MimeType.JSON);
}

// Cho phép dashboard xem lại danh sách + nội dung các file đã lưu, không cần mở Drive thủ công.
// GET .../exec                → liệt kê tất cả file (đọc từ tên file, không mở từng file nên nhanh)
// GET .../exec?action=get&id= → trả về nguyên nội dung JSON của 1 file (data thô + summary)
function doGet(e) {
  var FOLDER_ID = 'DÁN_FOLDER_ID_VÀO_ĐÂY';
  var folder = DriveApp.getFolderById(FOLDER_ID);
  var action = (e && e.parameter && e.parameter.action) || 'list';

  if (action === 'get') {
    var id = e.parameter.id;
    if (!id) {
      return ContentService.createTextOutput(JSON.stringify({status:'error', message:'Thiếu tham số id.'}))
        .setMimeType(ContentService.MimeType.JSON);
    }
    var file = DriveApp.getFileById(id);
    var content = file.getBlob().getDataAsString();
    // trả thẳng nội dung JSON gốc của file (đã có sẵn cấu trúc {dateStr, timeStr, task, fileName, employees, summary})
    return ContentService.createTextOutput(content).setMimeType(ContentService.MimeType.JSON);
  }

  // action === 'list': chỉ đọc TÊN file (không mở nội dung) để nhanh kể cả khi có rất nhiều file lịch sử.
  var files = folder.getFilesByType(MimeType.PLAIN_TEXT);
  var out = [];
  while (files.hasNext()) {
    var f = files.next();
    var name = f.getName();
    var m = /^productivity_(pick|pack)_(\d{4}-\d{2}-\d{2})_([\d-]+)\.json$/.exec(name);
    if (!m) continue; // bỏ qua file không đúng định dạng do doPost tạo ra
    out.push({
      id: f.getId(),
      fileName: name,
      task: m[1],
      dateStr: m[2],
      timeStr: m[3].replace(/-/g, ':')
    });
  }
  return ContentService.createTextOutput(JSON.stringify({status:'ok', files: out}))
    .setMimeType(ContentService.MimeType.JSON);
}
